    Represents Free Value View that is backed by a (usually small) dynamically allocated block.
  • pvr::GenericExtent3D
    Contains a 3D Integer size (width, height, depth)
  • pvr::TPSOrbitCamera
    A simple third person orbit camera implementation
  • pvr::GeneratePixelType1
    Use this template class to generate a 1 channel PixelID.
  • pvr::BlendingConfig
    Add blending configuration for a color attachment. Some API's only support one blending state for all attachments, in which case the 1st such configuration will be used for all.
  • pvr::GenericOffset3D
    The GenericOffset3D contains the offsets in 3 dimension (offsetX, offsetY, offsetZ)
  • pvr::ImageSubresourceLayers
    Represents a specific subresource layer: A specified Array Layer and Mipmap level of specific aspect of an image
  • pvr::ImageLayersSize
    Structure describes the number of array levels and mip levels an image contains
  • pvr::DisplayAttributes
    Contains display configuration information (width, height, position, title, bpp etc.).
  • pvr::GeneratePixelType2
    Use this template class to generate a 2 channel PixelID.
  • pvr::ImageResolveRange
    Represents an image resolve operation
  • pvr::IndexOutOfRange
    A simple std::runtime_error wrapper for throwing exceptions when attempting to use index out of range
  • pvr::LockedQueue
    A simple adapter for the BlockingConcurrentQueue that simplifies common use cases
  • pvr::ImageSubresourceRange
    Represents a subresource range: A specified range of Array Layers and Mipmap levels of specific aspect of an image
  • pvr::FPSCamera
    A simple first person camera implementation
  • pvr::Dereferenceable
    DO NOT USE DIRECTLY. The RefCountedResource follows this class.
  • pvr::FileNotFoundError
    A simple std::runtime_error wrapper for throwing exceptions when file not found errors occur
  • pvr::UnsupportedOperationError
    A simple std::runtime_error wrapper for throwing exceptions when unsupported operations are attempted
  • pvr::EmbeddedRefCount
    EmbeddedRefCount is the class that must be inherited from in order to have a class that automatically has refcounted members, with awareness and access to their ref counting by keeping their ref-counting bookkeeping entries embedded in the main class
  • pvr::RefCountedWeakReference
    A RefCountedWeakReference is a "Weak Reference" to a Reference counted object.
  • pvr::IRefCountEntry
    An interface that represents a block of memory that will be doing the bookkeeping for an object that will be Reference Counted. This bit of memory holds the reference counts.
  • pvr::GenericOffset2D
    The GenericOffset2D contains a 16-bit 2D offset (offsetX, offsetY)
  • pvr::FileStream
    A FileStream is a Stream that is used to access a File in the filesystem of the platform.
  • pvr::InvalidOperationError
    A simple std::runtime_error wrapper for throwing exceptions when invalid operations are attempted
  • pvr::GeneratePixelType4
    Use this template class to generate a 4 channel PixelID.
  • pvr::EmbeddedRefCountedResource
    "Embedded" subset of the Reference counted smart pointer. See RefCountedResource. It can be used very similarly to the C++11 shared_ptr, but with some differences, especially as far as conversions go. The EmbeddedRefCountedResource is used when the user must not have access to the .construct(...) functions of the RefCountedResource (especially, when a class is designed around reference counting and cannot stand widthout it). A RefCountedResource is-a EmbeddedRefCountedResource and is safe to "slice" to it as well (see details).
  • pvr::VertexAttributeLayout
    This class contains all the information of a Vertex Attribute's layout inside a block of memory, typically a Vertex Buffer Object. This informations is normally the DataType of the attribute, the Offset (from the beginning of the array) and the width (how many values of type DataType form an attribute).
  • pvr::RefCountEntry
    DO NOT USE DIRECTLY. The RefCountedResource uses this class when required. A "simple" Refcount entry keeps a user-provided pointer to the object together with the reference counts.
  • pvr::Texture
    A 2D Texture asset, together with Information, Metadata and actual Pixel data. Only represents the actual data, not the API objects that may be created from it.
  • pvr::Rectangle
    A class representing an axis-aligned rectangle. Internal representation TopLeft and size.
  • pvr::GenericExtent2D
    Contains a 2d Integer size (width, height)
  • pvr::ImageRange
    Represents a specific 3-D range in an image (an orthogonal cuboid anywhere in the image)
  • pvr::ImageSubresource
    Describes a single "layer" of an image: a single array layer of a single mip level, or the offset of a layer range.
  • pvr::hasher_helper<hashvalue,first>
    Template specialization for uint32_t values.
  • pvr::CameraKeyFrame
    A simple structure for handling key camera frames
  • pvr::hash<uint64_t>
    Template specialization of hash specifically for uint64_t
  • pvr::ImageStorageFormat
    Extends the ImageDataFormat with mipmaps and number of Samples.
  • pvr::CompressedImageDataFormat
    Describes a Compressed format. Compressed formats provide less information than the uncompressed format, as they can only be accessed "black box".
  • pvr::Rectangle3D
    A class representing an Axis-Aligned cuboidal region of 3D space. Internal representation offset defined as the absolute coordinates of one corner of the bounding region and extent which defines the second corner of the bounding region relative to the offset.
  • pvr::FilePath
    Filepath represents a Path + Filename + Extension.
  • pvr::Time
    This class provides functions for measuring time: current time, elapsed time etc. High performance timers are used if available by the platform.
  • pvr::BufferStream
    This class is used to access a block of memory as a Stream.
  • pvr::hash<uint32_t>
    Template specialization of hash specifically for uint32_t
  • pvr::Stream
    This class is used to abstract streams of data (files, blocks of memory, resources etc.). In general a stream is considered something that can be read or written from. Specializations for many different types of streams are provided by the PowerVR Framework, the most commonly used ones being Files and Memory. The common interface and pointer types allow the Stream to abstract data in a very useful manner.
  • pvr::TextureHeader
    A class mirroring the PVR Texture container format header, and which can in general represent any Texture asset. Contains accessors functions to facilitate using the Texture data in application code.
  • pvr::TextureDecompressionError
    A simple std::runtime_error wrapper for throwing exceptions when invalid operations are attempted
  • pvr::WindowsResourceStream
    A Stream implementation that is used to access resources built in a MS Windows executable.
  • pvr::FileEOFError
    A simple std::runtime_error wrapper for throwing exceptions when attempting to read past the End Of File
  • pvr::TPSCamera
    A simple third person camera implementation
  • pvr::PaletteExpander
    Internally used by some texture readers.
  • pvr::hasher_helper<hashvalue,first,dummy>
    Template specialization for uint32_t values.
  • pvr::InvalidDataError
    A simple std::runtime_error wrapper for throwing exceptions when invalid data is provided
  • pvr::InvalidArgumentError
    A simple std::runtime_error wrapper for throwing exceptions when invalid arguments are provided
  • pvr::hash<int32_t>
    Template specialization of hash specifically for int32_t
  • pvr::HashCompileTime
    Template specialization for unsigned char values.
  • pvr::hash
    Class template denoting a hash. Specializations only - no default implementation. (int32_t/int64_t/uint32_t/uint64_t/string)
  • pvr::RefCountedResource
    Reference counted smart pointer. It can be used very similarly to the C++11 shared_ptr, but with some differences.
  • pvr::FreeValueView
    Represents a runtime-known number of elements of a runtime-known type, with functions to handle and convert them. Very commonly used to dynamically represent data that will eventually need to be used by the GPU, such as uniforms, vectors and matrices. Does not contain methods to actually be populated, as this should be done through subclasses.
  • pvr::hash<int64_t>
    Template specialization of hash specifically for int64_t
  • pvr::TextureMetaData
    The TextureMetaData class contains metadata of a texture. Metadata is any information that a texture could be correctly loaded from file without. In most cases, metadata may still be necessary to actually USE the texture, such as winding orders, paddings, atlas information and others.
  • pvr::RefcountEntryHolder<void>
  • pvr::GeneratePixelType3
    Use this template class to generate a 3 channel PixelID.
  • pvr::Plane3d
    Uses the plane equation Ax + By + Cz + D = 0, where A B C are plane normal, xyz are position on the plane and D is distance to the plane.
  • pvr::StridedBuffer
    Representation of raw data. Used to store raw data that is logically grouped in blocks with a stride.
  • pvr::SwizzleChannels
    The SwizzleChannels struct
  • pvr::FileIOError
    A simple std::runtime_error wrapper for throwing exceptions when undertaking File IO operations
  • pvr::OperationFailedError
    A simple std::runtime_error wrapper for throwing exceptions when operations fail
  • pvr::StringHash
    Implementation of a hashed std::string with functionality for fast compares.
  • pvr::ImageStorageFormatCompressed
    Describes a Compressed format. Compressed formats provide less information than the uncompressed format, as they can only be accessed "black box".
  • pvr::ImageDataFormat
    Represents an image format, including pixel format(channels/bits per channel), datatype and colorspace.
